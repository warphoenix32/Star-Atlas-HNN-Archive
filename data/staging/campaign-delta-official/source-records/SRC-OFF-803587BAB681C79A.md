## Preserved Official Text

C4 PTR is a test environment, which means you may encounter bugs, unfinished systems, or features that change over time.
If you find an issue or want to share feedback, please use the in-game support menu so the team can review reports in the right place.
Open the support menu
From the main map, click the orange bug icon.
This opens the C4 PTR support menu.
From this menu, you can access support and feedback options, including:
Knowledge Base
Community Reports Dashboard
Bug reporting
Feedback and suggestions
Feature requests
Ticket creation
Before submitting a report
Before creating a new bug report or feedback entry, check the Community Reports Dashboard.
Another player may have already reported the same issue or suggested the same improvement.
If your issue already exists, you can upvote the existing report instead of creating a duplicate.
This helps the team understand which issues are affecting more players.
When to submit a bug report
Submit a bug report when something is not working as expected.
Examples include:
A transaction failed or did not complete correctly.
A button does not respond.
A menu does not open.
A fleet, resource, Claim Stake, or crafting action behaves incorrectly.
You receive an error message.
Something appears visually broken or missing.
When reporting a bug, try to include as much useful information as possible.
Helpful details include:
What you were trying to do
What happened instead
The steps needed to reproduce the issue
Screenshots or error messages, if available
Your browser and device information, if relevant
When to submit feedback or suggestions
Submit feedback or suggestions when something works, but you think it could be improved.
Examples include:
A menu is confusing.
A button or action is hard to find.
A flow feels too slow or unclear.
A tutorial, tooltip, or explanation is missing.
A feature could be easier to understand.
Feedback helps the team improve the player experience during PTR.
When to submit a feature request
Submit a feature request when you want to suggest a new feature or functionality that does not currently exist.
Feature requests are different from bug reports. A bug report is for something broken, while a feature request is for something new or missing.
When to create a support ticket
Create a support ticket if you need direct help from the support team.
Use this option when your issue requires personal support or account-specific investigation.
For general bugs, feedback, and feature requests, use the report and feedback tools first so the information can be tracked with other community reports.
Why reporting matters
The development team uses community reports to help track bugs and review feedback during PTR.
Using the support menu helps keep reports organized and makes it easier for the team to identify common issues, review player suggestions, and improve C4 PTR over time.
Summary
To report bugs or feedback in C4 PTR:
Open the main map.
Click the orange bug icon.
Check the Community Reports Dashboard first.
Upvote an existing report if your issue is already listed.
Submit a new bug report, feedback entry, feature request, or support ticket when needed.
