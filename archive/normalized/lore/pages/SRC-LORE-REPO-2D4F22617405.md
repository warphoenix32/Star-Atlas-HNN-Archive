## The Pavo Standard (Gold Standard)

Pavo Passage represents the deepest-developed sector in the canon. Its structure defines the **minimum depth target** for all Galia sectors.

### What Pavo Has That Others Don't

| Layer | Pavo Passage | Typical Sector |
|-------|-------------|----------------|
| **Sector overview** | ✅ 316-line comprehensive doc | ✅ 80–120 line overview |
| **Faction sheets** | ✅ 8 dedicated files (Crumon Dynasty, Order of Seasons, Ruling Conclave, MTC, VHE, TAPP, Winter Fist Path, Ustur Regency Pavo) | ❌ Factions mentioned inline only |
| **World sheets** | ✅ 6 worlds with 7–8 POIs each | ❌ No world-level breakdowns |
| **Species file** | ✅ Dedicated Crumon species doc | ❌ Species mentioned inline |
| **Per-world POIs** | ✅ 45 named POIs with control tags | ❌ Zero |
| **Economy per world** | ✅ Distinct economic roles per world | ❌ Sector-level economy only |
| **Travel hazards** | ✅ Per-world hazard lists | ❌ Generic sector hazards |
| **Named characters** | ✅ 27+ named individuals with roles | ⚠️ 1–3 key figures max |
| **Faction methods per world** | ✅ Each faction behaves differently per location | ❌ Single faction description |
| **Story fuel** | ✅ Per-world narrative tension | ⚠️ Sector-level hooks only |
| **Cross-references** | ✅ Tight web between worlds/factions | ⚠️ Loose references to other sectors |

### Why Pavo Works (Design Principles)

1. **Systemic worldbuilding**: Each world exists because the system *needs* it — not just for variety
2. **Functional differentiation**: Every world has a clear role (authority, throughput, proof, resources, opportunity, absence)
3. **Factional polymorphism**: Each faction adapts its methods to each world (MTC uses contracts in Tollhaven, credit traps in Okuto, escrow rails in Velloris)
4. **Bureaucratic horror**: The threat isn't monsters — it's "pending," erasure, and proof traps
5. **Hidden layers**: Beneath the machine, something ancient and religious (Iris Dusk, The Night, corridors that shouldn't connect)
6. **Emotional grounding**: The Disappeared Chapel, The Absent Queen, "grief still needs a place to stand"

## Enrichment Protocol (Per Sector)

When enriching a sector to Pavo-level depth, follow this sequence:

### Phase 1: Canon Sheets (Sector-Level)

1. **Sector Rewrite** — Expand overview to ~200–300 lines with governance, economy, history, factions
2. **Faction Sheets** — Create dedicated files for each faction with significant presence
3. **Species Sheet** — If sector has a native/dominant species not yet documented
4. **Open questions resolved** — Close all ⏸️ items or flag as deliberate ambiguity

### Phase 2: World Sheets

5. **World Inventory** — Define how many worlds/stations the sector has and what ROLE each plays
2. **World Sheets** — For each world:
   - Overview + environment/feel
   - Governance & faction presence (how each faction operates HERE specifically)
   - Economy & exports (distinct from other worlds)
   - 6–8 POIs with control tags
   - Travel hazards
   - Story fuel
3. **Cross-referencing** — Wire worlds to each other and to sector factions

### Phase 3: Narrative Depth

8. **Named characters** — Populate key positions with named individuals (min. 10 per sector)
2. **Inter-world dynamics** — Define how worlds depend on / conflict with each other
3. **Hidden layers** — Plant mysteries, sealed rooms, rumors, and "Schrödinger's dungeons"

## Metrics: What "Pavo-Level" Means (Quantified)

A sector is considered "Pavo-level" when it has:

- [ ] Sector overview doc ≥ 200 lines
- [ ] ≥ 3 dedicated faction sheets (sector-specific variants)
- [ ] ≥ 4 world sheets with distinct functional roles
- [ ] ≥ 30 named POIs across all worlds
- [ ] ≥ 15 named characters with defined roles
- [ ] Per-world faction presence tables (not just sector-level)
- [ ] Per-world economy (not duplicating sector economy)
- [ ] Per-world travel hazards
- [ ] ≥ 3 "hidden layer" elements (rumors, sealed areas, ambiguous threats)
- [ ] Tight cross-reference network between worlds and factions
